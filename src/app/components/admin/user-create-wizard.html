This wizard creates another Honcho Inspector user. Two role choices are available:

  • Standard user — can sign in, view peers / sessions / representations / conclusions, and run their own Honcho queries through profiles they own. No access to /admin.
  • Admin — everything above, plus /admin: user management, audit log, dashboard overview, and Honcho reachability probes. At least one admin must always exist; the last admin cannot be demoted or deleted.
Honcho MCP service

The new user will sign in with the Honcho MCP service configured for this deployment. The MCP block is the same one ~/.config/opencode/opencode.json advertises:

URL
https://mcp.honcho.cloudbsd.org/
Header
Authorization: Bearer <JWT>
Header
X-Honcho-User-Name: mlapointe

The new Inspector username becomes the Honcho peer identifier the new user uses on the workspace — match the X-Honcho-User-Name above for an existing identity, or pick any new one if Honcho will create it on first contact.
